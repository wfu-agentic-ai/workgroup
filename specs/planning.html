Workgroup Planning

Workgroup: WFU Agentic AI Workgroup Status: Planning Last Updated: 2026-02-06

Workgroup context

Basic information

  • Title: Agentic AI Tools in the CLI
  • Level: Faculty/staff professional development
  • Discipline: Cross-disciplinary (digital literacy, AI tools)
  • Prerequisites: None — assumes zero CLI experience

Target participants

  • Primary audience: WFU faculty and staff
  • Expected size: 8–15 active participants (rolling membership)
  • Background: Varies widely; most have no terminal experience
  • Diverse needs: Range of comfort with technology, varying schedules

Format and logistics

  • Frequency: Weekly meetings, ongoing (no fixed end date)
  • Format: In-person meetings + async web content
  • Meeting duration: ~60 minutes
  • Tools provided: Workgroup website with tutorials, session notes, and resources

Learning goals

Workgroup-level outcomes

  1. Participants can navigate the file system and execute basic commands in a terminal
  2. Participants can use Git and GitHub for version control
  3. Participants can configure their shell environment
  4. Participants can use Claude Code for agentic coding workflows
  5. Participants can access and configure AI APIs
  6. Participants can set up and use MCP servers
  7. Participants can author documents with Markdown and Quarto

Key competencies

  • CLI literacy: Comfort working in a terminal environment
  • Version control: Tracking changes and collaborating via GitHub
  • AI tool fluency: Using agentic AI assistants effectively
  • Reproducibility: Creating shareable, reproducible workflows

Topic progression map

The workgroup follows a progressive sequence, but participants who join late can catch up via tutorials on the website.

Foundation (first sessions)

  • What is a terminal? Why use one?
  • Navigating the file system (cd, ls, pwd)
  • Working with files and directories (mkdir, cp, mv, rm)
  • Reading and editing files from the command line
  • Understanding paths (absolute vs. relative)

Shell environment

  • Shell configuration files (.bashrc, .bash_profile, .zshrc)
  • Environment variables and $PATH
  • Aliases and shell functions
  • Package managers (Homebrew on macOS)

Version control

  • Git fundamentals (init, add, commit, log, diff)
  • Branching and merging
  • GitHub: remote repos, push/pull, collaboration
  • Pull requests and code review

AI tools

  • Installing and configuring Claude Code
  • Working with Claude Code in a project
  • API keys and authentication
  • MCP servers: what they are and how to use them
  • Building custom workflows with AI tools

Onboarding strategy

New participants who join after the workgroup has started:

  1. Self-paced tutorials: The website provides written tutorials covering all foundation topics
  2. Buddy system: Pair new participants with someone who’s been attending
  3. Office hours: Facilitator available for one-on-one help
  4. No penalty for gaps: Content is modular; missing sessions doesn’t block progress

Pedagogical approach

Instructional framework

  • Primary approach: Hands-on, task-driven learning
  • Session structure: Brief demo → guided practice → independent exploration
  • Scaffolding: Early sessions provide step-by-step instructions; later sessions give goals and let participants find the path
  • Peer learning: Participants help each other; facilitator is not the only source of knowledge

Content design

  • Modular organization: Each tutorial is self-contained
  • Progressive difficulty: Foundation → intermediate → advanced
  • Worked examples: Every concept is illustrated with a concrete task
  • Real tasks: Use participants’ actual work as practice material when possible

Self-assessment and feedback

There are no grades. Instead:

  • Check-your-understanding prompts at the end of tutorials (try this, verify that)
  • Session feedback: Quick verbal or written check-ins at end of meetings
  • Self-assessment questions: “Can you do X without looking at the tutorial?”
  • Peer feedback: Participants review each other’s work in Git/GitHub exercises

Resources and materials

Required tools

  • A Mac with Terminal.app (or iTerm2)
  • A GitHub account
  • A text editor (VS Code recommended but not required)
  • Claude Code (installed during an early session)

Workgroup-created content

Success metrics

Participant engagement

  • Attendance: Consistent participation (recognizing rolling membership)
  • Tutorial completion: Participants working through async content
  • Peer support: Participants helping each other in and outside meetings
  • Independent use: Participants using CLI tools in their own work

Content quality

  • Coverage: All planned tutorials published and up to date
  • Clarity: Content is accessible to beginners; feedback incorporated
  • Usability: Website navigation is intuitive; resources are findable

Community health

  • New member onboarding: New participants can get started independently
  • Retention: Participants continue attending over time
  • Skill growth: Observable progression in participant capabilities
